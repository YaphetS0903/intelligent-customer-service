# 智能客服可试运行上线验收清单

本清单用于把当前系统从“功能代码已具备”推进到“可试运行上线”。每一项都需要在真实部署环境中完成记录，建议由管理员在 `/admin/deploy` 导出上线报告后归档。

## 1. 环境与上线底座

- [ ] `AUTH_SECRET` 已配置为至少 24 位随机长字符串，不使用开发默认值。
- [ ] `APP_BASE_URL` 已改为员工可访问的服务器 IP 或域名，不再是 `localhost`。
- [ ] MySQL 连接正常，健康检查确认核心表结构完整。
- [ ] `RAG_PROVIDER=local_text` 或 OpenAI File Search 模式已按实际方案确认。
- [ ] 对话模型配置已通过 `/admin/settings` 的“测试对话模型”。
- [ ] 资料上传大小与 Nginx `client_max_body_size` 保持一致。
- [ ] `/admin/deploy` 中没有红色“需处理”检查项。

## 2. 第三方服务联调

- [ ] TTS：`/admin/settings` 的“TTS 语音试听”能返回并播放服务端音频。
- [ ] TTS：员工问答回答和培训讲稿播放优先使用服务端音频，失败时能降级浏览器朗读。
- [ ] OCR：`/admin/settings` 的“OCR 实际测试”上传图片或扫描件 PDF 后能返回文字预览。
- [ ] OCR：扫描件或图片资料能在 `/admin/documents` 上传入库，并生成可检索分片。
- [ ] 数字人：`/admin/settings` 的“数字人接口测试”能返回任务编号或视频地址。
- [ ] 数字人：管理员能在 `/admin/training` 提交课程视频任务，员工课程页能看到状态或播放器。

## 3. 统一身份认证

- [ ] 如启用 OIDC，身份平台回调地址已配置为 `{APP_BASE_URL}/api/auth/sso/callback`。
- [ ] 如启用 OIDC，首次企业登录能自动创建员工账号并同步邮箱、姓名、部门。
- [ ] 如启用 LDAP / AD，系统内密码失败后能使用企业目录账号登录。
- [ ] LDAP / AD 登录能同步邮箱、姓名、部门、岗位。
- [ ] 被禁用的系统账号不会被 OIDC 或 LDAP 自动重新启用。

## 4. 知识库与权限

- [ ] 至少创建 1 个正式知识库。
- [ ] 至少上传 3 份已发布、可检索资料。
- [ ] 资料默认草稿，经过“提交审核 -> 审核发布”后员工端才可检索。
- [ ] 部门、岗位、密级、单文档 ACL 均用员工账号验证过可见范围。
- [ ] 员工端无权限知识库访问会被阻止并记录安全事件。

## 5. 培训模块

- [ ] 至少 1 个 PPTX 课程已生成讲稿并发布。
- [ ] 员工端 `/training` 能看到已发布课程，未发布或归档课程不可见。
- [ ] 员工播放讲稿后能保存学习进度。
- [ ] 至少 1 名员工完成课程，管理员能看到完课率变化。
- [ ] 员工能提交课程测验并生成成绩记录。
- [ ] 正式考试支持单选、多选、判断题，未完成 100% 学习时不可开考。
- [ ] 考试次数、时限和及格分生效，答案不会在提交前返回员工端。
- [ ] 通过考试后能下载中文 PDF 证书，证书撤销后不可再下载。
- [ ] 必修课程发布、截止日期、未完课和逾期提醒可用且每日去重。

## 6. 工单与安全审计

- [ ] 员工能从对话页发起人工协助工单。
- [ ] 管理员能在 `/admin/insights` 查看工单，并更新状态、优先级、处理人和处理备注。
- [ ] 手机号、身份证号、银行卡号、邮箱、密钥等敏感信息会脱敏并记录审计。
- [ ] 提示词注入输入会生成安全事件。
- [ ] 连续风险事件会生成异常访问告警。
- [ ] 上线前高危或严重安全事件均已处理或标记忽略。

## 7. 自动化测试与发布

- [ ] 本地 `npm run typecheck` 通过。
- [ ] 本地 `npm run build` 通过。
- [ ] 本地 `npm run test:e2e` 通过。
- [ ] GitHub Actions CI 能执行 `npm ci`、typecheck、build、Playwright 回归。
- [ ] 部署流水线 secrets 已配置：`DEPLOY_HOST`、`DEPLOY_USER`、`DEPLOY_SSH_KEY`、`DEPLOY_PORT`、`DEPLOY_APP_DIR`。
- [ ] 服务器可通过 `scripts/deploy-local.sh` 完成部署更新。

## 8. 试运行放行标准

- [ ] `/admin/deploy` 部署得分不低于 80%。
- [ ] `/admin/pilot` 试运行验收得分不低于 80%。
- [ ] QA 至少运行 30 条高频问题，通过率不低于 80%。
- [ ] 无未处理红色检查项。
- [ ] 已导出上线报告和指标 CSV。
- [ ] 已配置 MySQL 定时备份，并验证至少一次恢复流程。
